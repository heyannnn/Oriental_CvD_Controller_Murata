# トラブルシューティングガイド
Oriental Motor CVD Controller - Murata

最終更新: 2026-03-05

---

## 基本情報

### システム構成
- **工程02 (マスター)**: キーボード操作可能、全工程を制御
- **工程03-11**: マスターからのOSC信号で動作
- **通信**: OSCプロトコル (UDP 10000番ポート)

### 基本操作
- **Vキー**: 開始/停止
- **Ctrl+V**: リセット (停止→アラームクリア→原点復帰→待機)
- **Ctrl+C**: 終了

## トラブルシューティング

### 【パターン1】1工程だけモニターが停止した

**症状:**
- 1つの工程のモニター映像がフリーズ/停止
- 他の工程は正常に動作

**原因:**
- ビデオプレイヤーのフリーズ
- OSC通信の一時的な問題
- Raspberry Piの負荷過多

**対処法:**

1. **ソフトリセット (推奨)**
   ```
   マスター(工程02)でCtrl+Vを押してリセット
   ```
   - 全工程が停止し、原点復帰後に待機状態に戻る
   - 問題のある工程も含めて全体がリセットされる

2. **該当工程のサービス再起動**
   ```bash
   # 該当工程のRaspberry Piにログイン後
   sudo systemctl restart oriental-controller
   ```
   - 再起動後、約5秒で準備完了
   - マスターから改めて開始操作が必要

3. **該当工程の再起動 (最終手段)**
   ```bash
   sudo reboot
   ```
   - 完全再起動には約30-60秒かかる

**注意:**
- 1工程だけの問題でも、通常は全体リセット (Ctrl+V) を推奨
- システムの同期を保つため

### 【パターン2】全工程のモニターが停止した

**症状:**
- すべての工程でモニター映像がフリーズ
- モーターは動作している可能性あり

**原因:**
- マスター(工程02)のビデオプレイヤー問題
- OSC信号の送信問題
- ネットワーク全体の問題

**対処法:**

1. **リセットボタン (Ctrl+V) を押す**
   ```
   工程02でCtrl+Vを押してリセット
   ```
   - 全工程が停止→原点復帰→待機に戻る

2. **マスターのサービス再起動**
   ```bash
   # 工程02で実行
   sudo systemctl restart oriental-controller
   ```

3. **ネットワーク確認**
   ```bash
   # 各工程への疎通確認
   ping 192.168.1.3
   ping 192.168.1.4
   # ... (工程11まで)
   ```
   - 疎通できない工程があれば、LANケーブル/スイッチを確認

4. **全工程の再起動 (最終手段)**
   ```bash
   # 各Raspberry Piで順次実行
   sudo reboot
   ```

### 【パターン3】モーターだけ停止した (メカ停止)

**症状:**
- モニター映像は再生されている
- モーターが動かない、または途中で停止

**原因:**
- **アラーム発生**: モーターが障害物に当たった、過負荷
- **HOMES検出**: 予期しない位置センサー反応 (工程05, 08, 09)
- **モーター通信エラー**: RS485通信の問題

**対処法:**

1. **自動リセット機能**
   - システムは自動的にアラームを検出しリセットを試みる
   - ログに `AUTO RESET` と表示される
   - 自動で原点復帰→運転再開

2. **手動リセット (自動復旧しない場合)**
   ```
   工程02でCtrl+Vを押してリセット
   ```

3. **物理的な確認**
   - モーター周辺の障害物を確認
   - ケーブルの接続を確認
   - センサーの状態を確認

4. **ログ確認**
   ```bash
   # 該当工程で実行
   tail -f /tmp/main_XX.log
   # (XXは工程番号: 02, 03, ... 11)
   ```
   - `ALARM detected` = アラーム発生
   - `HOMES detected` = 位置センサー誤検出
   - エラーメッセージから原因を特定

5. **アラームクリア後も停止する場合**
   ```bash
   # 該当工程のサービス再起動
   sudo systemctl restart oriental-controller
   ```

**工程別の注意:**
- **工程02, 03, 04, 10, 11**: 原点復帰不要 → 停止するだけで待機状態へ
- **工程05, 06, 07, 08, 09**: 原点復帰必要 → リセット時に原点復帰実行

### 【パターン4】1工程だけ反応しない

**症状:**
- 開始操作しても特定の工程だけ動作しない
- モニターにも変化なし

**原因:**
- OSC通信の受信失敗
- サービスが停止している
- ネットワーク問題

**対処法:**

1. **サービス状態確認**
   ```bash
   # 該当工程で実行
   sudo systemctl status oriental-controller
   ```
   - `active (running)` と表示されればOK
   - `inactive` や `failed` ならサービスが停止中

2. **サービス再起動**
   ```bash
   sudo systemctl restart oriental-controller
   ```

3. **ネットワーク確認**
   ```bash
   # マスター(工程02)から該当工程へping
   ping 192.168.1.X  # Xは工程番号
   ```

4. **ログ確認**
   ```bash
   tail -f /tmp/main_XX.log
   ```
   - `Received /start via OSC` が表示されない → OSC受信できていない

### 【パターン5】原点復帰に失敗する

**症状:**
- リセット操作後、原点復帰中にタイムアウト
- `Homing timeout` エラー

**原因:**
- 原点センサーの故障/汚れ
- モーター動作範囲外に位置
- メカニカルな問題

**対処法:**

1. **手動で原点付近に移動**
   - モーターを手で原点センサー位置まで移動
   - (モーター電源OFFの状態で可能)

2. **センサー確認**
   - 原点センサーのLED確認
   - センサー接続確認

3. **サービス再起動**
   ```bash
   sudo systemctl restart oriental-controller
   ```

4. **それでも失敗する場合**
   - モータードライバーの電源を一度OFFにして再起動
   - ハードウェアの点検が必要

   - リンクLEDの点灯確認

## 状態確認コマンド

### ログファイルの確認
```bash
# リアルタイムでログ監視
tail -f /tmp/main_XX.log  # XXは工程番号

# 最新100行を表示
tail -n 100 /tmp/main_XX.log

# エラーのみ抽出
grep ERROR /tmp/main_XX.log
grep ALARM /tmp/main_XX.log
```

### サービス状態確認
```bash
# 状態確認
sudo systemctl status oriental-controller

# 再起動
sudo systemctl restart oriental-controller

# 停止
sudo systemctl stop oriental-controller

# 開始
sudo systemctl start oriental-controller
```

### ネットワーク確認
```bash
# 全工程への疎通確認スクリプト
for i in {2..11}; do
  echo "Testing 192.168.1.$i..."
  ping -c 1 192.168.1.$i > /dev/null && echo "OK" || echo "FAILED"
done
```

## 状態の種類と意味

| 状態 | 意味 | 次のアクション |
|------|------|----------------|
| `booting` | 起動中 | 自動的に次の状態へ |
| `homing` | 原点復帰中 | 完了まで待機 |
| `home_end` | 待機中 (開始可能) | Vキーで開始 |
| `running` | 運転中 | 自動で完了へ |
| `ready` | サイクル完了 (ループ継続) | 自動で次サイクルへ |
| `error` | エラー発生 | Ctrl+Vでリセット |
| `resetting` | リセット中 | 完了まで待機 |

## よくある質問

### Q: リセットすると他の工程も止まりますか？
**A:** はい。Ctrl+Vを押すと全工程が停止→原点復帰→待機状態になります。これはシステム全体の同期を保つためです。

### Q: 運転中にリセットしても大丈夫ですか？
**A:** 大丈夫です。リセットコマンドは安全に停止→アラームクリア→原点復帰の順で実行されます。

### Q: 自動リセットとは何ですか？
**A:** アラームやHOMES検出時に、システムが自動的に停止→クリア→原点復帰→運転再開を行う機能です。ループ運転中に有効です。

### Q: どの工程のログを確認すればいいですか？
**A:** 問題が発生している工程のログを確認してください。ファイル名は `/tmp/main_XX.log` です (XXは工程番号)。

### Q: ネットワークが切れるとどうなりますか？
**A:** 各工程は独立して動作するため、通信が切れても動作中の処理は継続されます。ただし、新しい開始/停止コマンドは受信できません。
